เรื่องที่ 1
สวัสดีครับ Yoh วันนี้มีเรื่องใหญ่ที่ Pentagon เซ็นสัญญาให้ Google เปิด Gemini ใช้บน classified network ของกลาโหมแบบ all lawful uses เต็มรูปแบบ — ฟังเหมือนข่าว procurement ปกติ แต่จริง ๆ มันคือบทใหม่ของวงการ frontier model ที่กำลัง diverge เรื่องค่านิยมแล้ว ที่มาคือ Anthropic ปฏิเสธเงื่อนไขเดียวกันมาตั้งแต่ต้นปี Dario Amodei ยืนกรานต้องมี carve-out ว่าห้ามใช้ Claude ทำ mass surveillance พลเมืองอเมริกัน และห้ามคุม autonomous weapons โดยตรง Pentagon ตอบกลับว่า workflow defense มี gray area เยอะเกินกว่าจะ pre-define ได้ ปลาย ก.พ. รัฐมนตรีกลาโหม Hegseth ประกาศ Anthropic เป็น supply chain risk ระดับเดียวกับ Huawei ในปี 2019 — ครั้งแรกที่บริษัทอเมริกันโดน designation นี้ Anthropic ฟ้อง ศาลล่างบล็อก แต่ศาลอุทธรณ์ DC พลิกกลับเมื่อ 8 เม.ย. ตอนนี้ Anthropic ห้ามขาย DoD ระหว่างคดี ที่ Google เองมีพนักงาน 950 คนเซ็นจดหมายขอให้ทำตาม Anthropic แต่ผู้บริหารเดินหน้า signal สำคัญที่สุดคือ ค่ายโมเดลใหญ่กำลังเลือก side ชัดเจน OpenAI กับ Google ยอมเงื่อนไข lawful use Anthropic ไม่ยอม — และนี่กระทบ multi-cloud thesis ที่เพิ่งเปิดตัวสัปดาห์ก่อน เพราะลูกค้า enterprise ต้องเริ่มถามว่า model ที่เลือกใช้ได้กับ workflow defense ของลูกค้าตัวเองไหม สำหรับ OpenBridge — ไม่ direct เกี่ยว แต่ adjacent insight ใหญ่ vendor risk disclosure จะกลายเป็น procurement question แรก ๆ ที่ลูกค้า bank insurer healthcare ขนาดใหญ่ของไทยจะถามภายใน 60-90 วัน positioning ที่เราขายต้อง pivot จาก ขาย integration เป็น ขาย vendor neutrality กับ policy disclosure layer ใครเข้าตลาดก่อนจะมี compliance moat ขายได้ทั้งไทยและอาเซียน

เรื่องที่ 2
Yoh ครับ ที่ Appian World 2026 เมื่อ 28 เม.ย. มีประกาศใหญ่ที่หลายคนพลาดเพราะ Adobe กับ Google ดูดความสนใจไป Appian ที่เป็น BPM SaaS ขายเข้า Bank of America, U.S. Air Force, NHS UK ประกาศ adopt MCP แบบ first-class เลย agent ภายนอกเข้ามาที่ Appian data fabric ได้ผ่าน MCP server มาตรฐาน Appian agent ออกไปเรียก external tool ก็ผ่าน MCP เหมือนกัน CEO Matt Calkins บอกว่า third party AI agent ใครก็ได้จะเข้ามาใช้ Appian data fabric ได้ unified read-write ที่ใหญ่กว่านั้นคือ Appian ประกาศ partnership กับ Snowflake พร้อมกัน Appian = orchestration layer Snowflake = AI Data Cloud + Cortex AI เชื่อมกันผ่าน MCP-enabled connector ตรง agent ที่ทำงานใน Appian process มี enterprise context จาก Snowflake พร้อม governance lineage ครบ ฟีเจอร์ที่ฝั่ง developer ตื่นเต้นที่สุดคือ AI-assisted spec-driven development ที่ AI ดูดสเปคจาก legacy app เก่า Cobol .NET mainframe แล้ว generate visual plan UI data model process flow ที่ทีม IT migrate ได้เร็วขึ้นมาก และ MCP server สำหรับ dev tool — ทีมที่ใช้ Claude Code หรือ Kiro เปิด session แล้วสั่ง build Appian app ผ่าน chat ได้ตรง pattern ที่เห็นคือ MCP ขึ้นมาอยู่ที่ orchestration layer ของ Fortune 500 ละ Appian เลือก position ตัวเองเป็น protocol-neutral broker ตรงข้ามกับ ServiceNow ที่ build agent platform ของตัวเอง มุม OpenBridge เกี่ยวตรง — Appian + Snowflake คือ blueprint ที่ OpenBridge ต้อง mirror ใน mid-market ASEAN ลูกค้าไทยที่ไม่มี budget Appian Snowflake รวม 600 พันดอลลาร์ต่อปีจะหา regional alternative OpenBridge ต้อง ship MCP server first-class data fabric แบบเบา และ spec-driven build flow ใน 6-9 เดือนก่อน Appian ลง ASEAN เต็มตัว

เรื่องที่ 3
Yoh วันนี้มีเรื่อง vertical AI ที่กลายเป็น unicorn เร็วมาก Avoca ที่เป็น YC alum 2023 ระดมทุนรวมแล้ว 125 ล้านที่ valuation 1 พันล้านดอลลาร์ Series B นำโดย Meritech กับ General Catalyst Series A นำโดย Kleiner Perkins ที่ Avoca ทำคือสิ่งเดียว — รับโทรศัพท์แทนช่างประปา ช่าง HVAC ช่างซ่อม operator ของ services economy ทุกคน agent voice ทำงาน 24/7 รับ missed call นัดเวลา ส่ง follow-up SMS รัน marketing campaign autonomous เจ้าของกิจการที่เคยพลาด lead 80% ตอนกลางคืน ตอนนี้รับได้ 100% ตัวเลขที่เด่นที่สุดคือ Avoca อยู่ใน track ที่ลูกค้าจะ booking งานรวมกัน 1 พันล้านดอลลาร์ปีนี้ — ไม่ใช่ revenue ของ Avoca แต่เป็นมูลค่างานที่ลูกค้าได้ คุณ tyler chen founder เล่าว่า idea นี้มาจากเพื่อนที่เป็นช่างประปาใน Texas บ่นว่าพลาด lead ทุกวัน ที่น่าสังเกตคือ Avoca ไม่ build foundation model ของตัวเอง — ใช้ Vapi Retell ElevenLabs เป็น infrastructure ขายเป็น vertical knowledge + sales motion เป็น moat ตลาด AI ส่วนใหญ่อ่านข่าวนี้ผ่าน เพราะช่างประปาดูเล็กเทียบ Anthropic ไม่ได้ — แต่จริง ๆ Avoca เป็น proof point ของ thesis ที่ a16z เขียนไว้ vertical agent สำหรับ SMB ที่ไม่เคยซื้อ enterprise software คือตลาด greenfield ที่ใหญ่กว่า horizontal tool ของยักษ์ pattern คือ vertical agent กิน workflow ที่ Salesforce ServiceNow เข้าไม่ถึง — TAM 850000 service operator ในสหรัฐมูลค่าต่อ customer ต่ำพอที่ยักษ์ไม่สนแต่สูงพอ Avoca อยู่ได้กำไรงาม สำหรับ OpenBridge ใกล้เคียงมาก ตลาด vertical voice agent ของไทยอาเซียนยังเป็น greenfield สมบูรณ์ Avoca เข้ายากเพราะภาษาวัฒนธรรม payment integration ต่างหมด OpenBridge มีสองทาง — build vertical agent ของตัวเองสำหรับ HVAC ขนของย้ายในไทย หรือเป็น orchestration layer ให้ vertical agent ของคนอื่น ผมแนะ path 2 บวก selective path 1 อีกประเด็น Avoca ระดมทุนได้โดยไม่ build model ของตัวเอง — VC ใหญ่ระดับ Meritech ยอมเข้า แปลว่า founder ASEAN ควร rethink thesis ที่ขาย Thai LLM ไปเป็นชนะ vertical workflow แทน

เรื่องที่ 4
Yoh วันนี้มีเรื่อง physical AI จากเยอรมัน Sereact ระดม Series B 110 ล้านดอลลาร์ นำโดย Headline ทำให้ funding รวม 140 ล้าน Series B ใหญ่กว่า Series A กว่า 4 เท่า Sereact ขายไม่ใช่หุ่นยนต์ฮาร์ดแวร์ — เป็น software platform ทำให้หุ่นยนต์ของลูกค้าฉลาดพอที่จะหยิบของในคลังสินค้าที่ของไม่เรียงเป็นระเบียบ ลูกค้าหลักรวม BMW PepsiCo deploy ไปแล้วกว่า 200 ระบบทั่วยุโรป ตัวเลขที่ทึ่งที่สุดคือทำ production picks สะสมเกิน 1 พันล้าน pick และ human takeover rate อยู่ที่ 1 ใน 53000 pick — เทียบคู่แข่งทั่วไปอยู่ที่ 1 ใน 500-2000 pick ทำให้ ROI ของ warehouse automation เปลี่ยน 5-10 เท่า จุดเปลี่ยนคือ Cortex 2.0 ที่ปล่อยพร้อมระดมทุน — เพิ่ม world model เข้าไปใน vision-language-action baseline หุ่นยนต์รัน possible action ผ่าน learned physics simulation ก่อนเคลื่อน CEO Ralf Gulde บอกว่า VLA model อย่างเดียว hit ceiling เพราะ react แต่ predict ไม่ได้ — เพิ่ม world model ทำให้ robot คิดก่อน act pattern เดียวกันที่ Pi Skild DeepMind ทุกค่ายกำลังเดินตาม Sereact จะเปิด US office ที่ Boston summer 2026 มุม OpenBridge ไม่ direct เกี่ยวเพราะอยู่ software layer แต่ adjacent insight สามข้อสำคัญมาก หนึ่ง human takeover rate จะถูกย้ายเข้า software agent world ใน 6-12 เดือน enterprise จะเริ่มถามใน RFP ว่า agent ของคุณให้คน intervene กี่ครั้งต่อ 1000 task OpenBridge ควรมี takeover rate metric เป็น first-class observability feature ตอนนี้ สอง world model + sim-loop pattern จะถูก port เข้า software agent — agent ที่ดีกว่าจะ simulate consequence ก่อน act เช่น agent procurement simulate negotiation ก่อนส่ง offer สาม physical AI กับ software agent deployment จะรวมกันเร็ว ลูกค้า logistics ไทย Kerry Flash J&T ที่ deploy Sereact-class robot จะมี optionality ที่ OpenBridge เชื่อม physical agent + software agent เข้า workflow เดียวกัน — ตลาดนี้ยังไม่มีใครจับ ASEAN
